2. Data 기반 중앙처리식 액터 행동 제어 구조 구축
   
---

# 설계 방향성
## 컴포넌트 기반 액터 아키텍처
- 'Actor' 추상 클래스는 최소한의 데이터와 생명주기만 관리하는 논리적 주체로 정의
- 이동, 공격 등 개별 기능을 별도의 컴포넌트로 분리하여 필요에 따라 유연하게 조합하도록 설계

## 인터페이스 기반의 행동 확장성
- 모든 액터의 행동은 'IAction' 인터페이스를 상속받도록 규격화하여 일관되게 관리
- 코드 수정 없이 새로운 행동을 쉽게 추가할 수 있는 구조 설계

## 신기술 R&D 고려
- Unity ECS 고려 : Data-Driven 설계를 적용하여 추후 Unity ECS 전환이 용이하도록 데이터 분리
- Dedicated Server 고려 : Unity 6의 데디케이트 서버 도입 시 서버에서 액터 연산을 제어할 수 있도록 로직과 뷰 분리
